## Patterns Identified

1. **Scope drift in 2.2 → 2.3:** Phase 2 of 2.2 absorbed 2.3's entire MCP client scope (agentic loop, tool-use handlers, Chainlit MCP lifecycle). Story 2.3 became validation + minor config improvements.
2. **⚠️ Critical Context sections worked:** Used in 2.3, 2.5, 2.6 — prevented agents from re-implementing existing code. Zero regressions across 6 stories.
3. **Dual MCP connection pattern (tech debt):** Chainlit native lifecycle handlers (2.2/2.3) vs. manual AsyncExitStack reconnect in 2.6's `on_chat_resume`. Two parallel patterns in codebase.
4. **Copilot PR review workflow effective:** Copilot caught real issues on every PR (missing `Field(ge=1)`, UUID in error handler, import issues). Some cosmetic comments dismissed appropriately.
5. **BMAD integration broken during Epic 2:** Plugin `state.json` and `config.yaml` were out of sync with actual project progress. Diagnosed and fixed during this retrospective session.
6. **No local dev scripts:** Starting/stopping MCP server, Chainlit, and Docker required manual multi-terminal setup. Slowed PR reviews significantly, especially for 2.6 (schema changes required `docker compose down -v`).

## Retrospective Meta-Note

This retrospective itself surfaced a significant process issue: the BMAD OpenClaw integration was misconfigured, causing the methodology to be bypassed during Epic 2 development. The plugin's `state.json` showed "analysis phase, zero workflows completed" while the project was in implementation with 2 epics done. Root cause: `bmad_init_project` was run on an already-initialized project, creating fresh state that overwrote progress. Fixed during this session by manually syncing `_bmad/config.yaml` and `_bmad/state.json` to reflect actual project state.
